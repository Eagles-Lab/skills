# SRE 简历最佳实践

本文档总结了 SRE 岗位简历的编写最佳实践,重点关注关键原则、常见错误和针对性建议。

---

## 一、关键原则

### 1. 量化成果

**不要写**:

```
- 优化了系统性能
- 提高了监控覆盖率
- 改进了告警系统
```

**要写**:

```
- 系统响应时间从 500ms 降低到 100ms (↓80%)
- 监控覆盖率从 30% 提升到 99.5% (↑69.5%)
- 告警准确率提升 60%,误报率降低 70%
- 系统可用性从 99.5% 提升到 99.95%
```

### 2. 突出技术栈

**不要写**:

```
- 使用监控工具搭建了监控平台
- 用容器技术部署了应用
```

**要写**:

```
- 使用 Prometheus + Grafana + Alertmanager 搭建监控平台
- 使用 Docker + Kubernetes 部署微服务,实现自动化扩缩容
```

### 3. 具体而非抽象

**不要写**:

```
- 负责系统运维工作
- 参与了监控系统的建设
```

**要写**:

```
- 负责 50+ 生产服务器的运维,保障系统 99.9% 可用性
- 主导监控系统选型和实施,部署 Prometheus 集群,监控 100+ 服务实例
```

### 4. 动词开头,主动语态

**推荐动词**:

- 设计 / 实施 / 搭建 / 部署 / 优化
- 开发 / 编写 / 配置 / 维护 / 管理
- 主导 / 负责 / 推动 / 改进 / 提升

**示例**:

```
✅ 设计并实施了完整的监控解决方案
✅ 开发了 10+ Python 自动化脚本
✅ 配置了 50+ 条告警规则

❌ 参与了监控系统的设计(被动)
❌ 使用了 Python 进行开发(模糊)
```

### 5. 关键词优化

**SRE 核心关键词**:

- 监控: Prometheus, Grafana, SLI/SLO, metrics
- 告警: Alertmanager, on-call, runbook
- 自动化: CI/CD, Terraform, Ansible, scripting
- 容器: Docker, Kubernetes, microservices
- 故障处理: incident response, postmortem, RCA
- 可靠性: high availability, disaster recovery

**合理使用,避免堆砌**:

```
✅ 使用 Prometheus + Grafana 搭建监控平台,定义 SLI/SLO,
   实现了 99.95% 的系统可用性

❌ 熟悉 Prometheus Grafana Kubernetes Docker Terraform Ansible
   Jenkins GitLab CI...(简单罗列,无上下文)
```

## 二、常见错误

### ❌ 错误 1: 内容空洞

**问题示例**:

```
项目: 监控系统建设
- 负责监控系统的搭建和维护
- 使用了相关工具
- 完成了任务
```

**如何改进**:

```
项目: 微服务监控平台建设
- 使用 Prometheus + Grafana 搭建监控平台,覆盖 100+ 服务实例
- 开发 Custom Exporter 10+ 个,采集业务指标
- 配置告警规则 50+ 条,告警准确率提升 60%
```

### ❌ 错误 2: 缺少量化

**问题示例**:

```
- 优化了系统性能
- 提升了监控覆盖率
- 改进了告警系统
```

**如何改进**:

```
- 系统响应时间从 500ms 降低到 100ms (↓80%)
- 监控覆盖率从 30% 提升到 99.5%
- 告警准确率提升 60%,误报率降低 70%
```

### ❌ 错误 3: 技术栈不明确

**问题示例**:

```
- 使用监控工具搭建了监控平台
- 用容器技术部署了应用
```

**如何改进**:

```
- 使用 Prometheus + Grafana 搭建监控平台
- 使用 Docker + Kubernetes 部署微服务
```

### ❌ 错误 4: 过度包装

**问题示例**:

```
项目: 企业级云原生微服务架构监控平台设计与实施
- 主导设计了世界级的监控系统
- 实现了 100% 的监控覆盖率
```

**如何改进**:

- 诚实描述,避免夸大
- 用数据说话,而非形容词
- 符合学生身份,不要过度包装

### ❌ 错误 5: 格式混乱

**问题**:

- 字体大小不一致
- 对齐方式混乱
- 缩进不规范
- 段落间距不一致

**解决方案**:

- 使用简历模板
- 统一字体(推荐:微软雅黑/Arial)
- 统一字号(正文 10-12pt,标题 14-16pt)
- 保持一致的间距和对齐

### ❌ 错误 6: AI 经验空洞或过度包装

**问题示例**:

```
❌ 熟悉人工智能、机器学习、深度学习
❌ 使用过 ChatGPT,了解大模型
❌ 学习过 AI 相关课程
❌ 用 AI 开发了智能系统(无具体描述)
```

**如何改进**:

```
✅ 使用 TensorFlow 构建时序预测模型
   - 预测系统 CPU/内存使用趋势
   - 准确率 85%,提前 7 天预警容量瓶颈
   - 避免了 3 次潜在故障

✅ 基于 LLM 开发智能运维助手
   - 使用 LangChain + Claude API 构建
   - 自动分析日志和指标,生成故障诊断建议
   - 平均故障定位时间从 30 分钟缩短到 10 分钟

✅ 开发 AI 巡检 Agent
   - 基于 GPT-4 API 构建,7x24 小时自动巡检
   - 发现并自动处理 80% 的常见问题
   - 运维人员工作负担降低 50%
```

## 三、针对 SRE 岗位的特殊建议

**示例**:

```

- 参与 On-call 轮值,处理生产环境故障 20+ 次
- 编写 Postmortem 报告,推动系统性改进
```

**示例**:

```

```
