# SRE 简历评分细则 (Scoring Rubric)

本文档详细定义了 SRE 简历六大评分维度的具体标准和评分细则。

---

## 评分原则

1. **证据驱动**: 所有评分必须基于简历中的具体证据
2. **一致性**: 同等质量的简历应获得相近评分
3. **成长导向**: 考虑学生身份,对学习能力和成长潜力给予适当权重
4. **实用性**: 避免过度理想化,关注实际可操作性
5. **AI 加分**: AI 相关经验(LLM、AI Agent、AI IDE、AIOps 等)作为重要加分项,体现技术前瞻性和创新能力

### AI 相关经验评分指导

**AI 技术在 SRE 场景的应用可作为重要加分项**:

- **智能监控和告警** (+1-2分):
  - 使用机器学习进行异常检测
  - 基于时序预测的容量规划
  - 智能告警降噪和聚类

- **AI 辅助运维** (+1-2分):
  - 基于 LLM 的智能运维助手
  - AI Agent 自动化巡检和故障诊断
  - 自动化 Runbook/Postmortem 生成

- **开发效率提升** (+0.5-1分):
  - 熟练使用 AI IDE (Cursor/Copilot)
  - 利用 AI 辅助代码生成和优化
  - prompt engineering 能力

- **AIOps 实践** (+1-2分):
  - 完整的 AIOps 平台建设经验
  - 预测性维护系统
  - 智能根因分析

**注意**: AI 经验需要结合具体 SRE 场景,单纯罗列 AI 工具而无实际应用场景的不应加分。

## 六大维度详解

## 总分计算与等级划分

### 总分公式

```
总分 = (监控得分 × 0.20) +
       (告警得分 × 0.15) +
       (自动化得分 × 0.20) +
       (容器化得分 × 0.15) +
       (故障处理得分 × 0.15) +
       (简历质量得分 × 0.15)
```

### 等级划分

| 总分范围 | 等级 | 描述 | 建议 |
|---------|------|------|------|
| 8.5 - 10.0 | A (优秀) | 简历质量优秀,可直接投递 | 保持优势,持续提升 |
| 7.0 - 8.4 | B (良好) | 简历质量良好,小幅优化即可 | 针对性改进薄弱项 |
| 5.5 - 6.9 | C (中等) | 简历质量中等,需要较大优化 | 重点关注高优先级改进项 |
| 4.0 - 5.4 | D (需改进) | 简历质量一般,需要大幅优化 | 建议全面重构简历 |
| < 4.0 | F (需大幅改进) | 简历质量较差,建议重新准备 | 从零开始重新编写 |

### 百分位估算

基于历史数据估算该简历在同类候选人中的位置:

- **Top 10%**: 总分 ≥ 8.5
- **Top 30%**: 总分 ≥ 7.0
- **Top 50%**: 总分 ≥ 6.0
- **Top 70%**: 总分 ≥ 5.0
- **Top 90%**: 总分 ≥ 4.0

## 特殊情况处理

### 缺失维度

如果简历完全未提及某个维度:
- **评分**: 1分
- **备注**: 简历中未体现相关经验,建议补充

### 争议处理

当评分存在争议时:
1. 查看具体证据
2. 参考多个评分标准
3. 倾向于给中间分
4. 在报告中注明争议点

### 异常高分

如果某个维度评分异常高(如9-10分):
1. 仔细验证证据真实性
2. 确认项目规模和复杂度
3. 检查是否与学生身份匹配
4. 必要时降低评分
